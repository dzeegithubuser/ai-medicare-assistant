# Chapter 10.9 — Error Handling, Backend-Only & Infrastructure

> Covers error handling & edge cases, CMS API (backend-only), clinical intelligence (backend-only), chatbot orchestrator (removed), and Serilog MongoDB logging.

## 10. Error Handling & Edge Cases

| # | Scenario | Trigger | Expected Result |
|---|----------|---------|-----------------|
| 10.1 | Network error | Kill backend → submit | **Error popup:** "Unable to reach the server. Please check your internet connection and try again." with technical details (status 0). |
| 10.2 | OpenAI failure | Invalid API key | **Error popup:** "An unexpected server error occurred. Please try again later." (status 500). Backend logs full error via GlobalExceptionMiddleware. |
| 10.3 | RxNorm timeout | RxNorm slow | Old analysis pipeline completes with AI-only data. (Backend-only) |
| 10.4 | CMS timeout | CMS slow | Old analysis pipeline renders without CMS data. (Backend-only) |
| 10.5 | API timeout | Financial Planner API slow | **Error popup:** "The request timed out. Please try again." (status 408/504). Loading spinner stops first (httpLoaderInterceptor finalize runs before httpErrorInterceptor). |
| 10.6 | AI pricing failure | IChatClient fails | Fallback to ParsePriceString prices. |
| 10.7 | Empty prescription | Submit empty text | Nothing happens (ignored). |
| 10.8 | Very long prescription | 20+ drug names | All valid drugs analyzed. Results rendered. |
| 10.9 | Unauthorized access | Access `/` without JWT | Redirected to `/signin`. |
| 10.10 | Global error popup — 401 API | JWT expired, call any API | **Error popup:** "Your session has expired. Please sign in again." |
| 10.11 | Global error popup — 403 API | Forbidden endpoint | **Error popup:** "You do not have permission to perform this action." |
| 10.12 | Global error popup — 429 rate limit | Rapid API calls trigger rate limit | **Error popup:** "Too many requests. Please wait a moment and try again." |
| 10.13 | Auth error handled inline | Invalid login credentials | **No popup.** Auth URLs excluded from interceptor. Inline error shown in sign-in form. |
| 10.14 | Popup deduplication | Two API calls fail simultaneously | Only one error popup shown. Second error silently re-thrown to subscriber. |
| 10.15 | Error popup technical details | Click "Technical details" on popup | Expandable section shows: `POST https://…/api/drug/analyze`, `Status: 500 Internal Server Error`. |

## 7. Medicare Cost Data — CMS API (Backend-Only, Not Displayed in UI)

> **Note:** CMS enrichment was part of the removed `POST /api/drug/analyze` pipeline. The current UI flow does not render CMS cost cards for individual drugs. Plan-level cost breakdowns (from the Financial Planner API) are shown instead. These scenarios are retained for reference only.

| # | Scenario | Input | Expected Result |
|---|----------|-------|------------------|
| 7.1 | Drug found | `"Eliquis 5mg"` via `/api/drug/analyze` | CMS data included in API response with source, data year, costs. |
| 7.2 | Drug not found | `"Famciclovir 250mg"` via `/api/drug/analyze` | CMS section empty in response. AI estimates present. |
| 7.3 | CMS timeout | Any drug, CMS slow | Response completes without CMS data. No errors. |

## 8. Clinical Intelligence (Backend-Only, Not Displayed in UI)

> **Note:** Clinical intelligence fields (alternatives, generic switch, contraindications, confidence score) were generated by the removed `POST /api/drug/analyze` AI pipeline and are not rendered by the current UI. No drug card components exist. These scenarios are retained for reference only.

| # | Scenario | Input | Expected Result |
|---|----------|-------|------------------|
| 8.1 | Therapeutic alternatives | `"Eliquis 5mg"` via `/api/drug/analyze` | Response includes alternatives: Warfarin, Xarelto. |
| 8.2 | Generic switch | `"Lipitor 20mg"` via `/api/drug/analyze` | Response includes: "Lipitor → Atorvastatin, savings $X/year". |
| 8.3 | Contraindications | `"Eliquis 5mg"` via `/api/drug/analyze` | Response includes contraindications: "Active pathological bleeding", etc. |
| 8.5 | Confidence score | `"Metformin 500mg"` via `/api/drug/analyze` | Response includes confidence score value. |

## 18. ~~Chatbot Orchestrator~~ (Removed)

> **Note:** The chatbot orchestrator (`ChatOrchestratorService`, `ChatOrchestratorController`, `ConvStateService`, `ConvStateDocument`, `DeltaCalculationService`, `DeltaDisplayComponent`, `HelpMenuComponent`) has been fully removed from the codebase. Chat coordination is now handled by `ChatRouterService` with `ChatIntentService` (20 intents), page-specific extraction services, and `ChatNavigationFlowService`. The `convStates` MongoDB collection no longer exists. All test scenarios in this section are obsolete.

## 22. Serilog MongoDB Logging

| # | Scenario | Steps | Expected Result |
|---|----------|-------|-----------------|
| 22.1 | Logs written to MongoDB | Start app, trigger any API request | `logs` collection in MongoDB receives structured BSON log entries within 5 seconds (batch period). |
| 22.2 | MinimumLevel filtering | Trigger EFCore debug-level log | Not written to MongoDB (override: Warning). Only Warning+ from EFCore appear. |
| 22.3 | File fallback | Stop MongoDB, start app | Logs written to `Logs/log-{date}.txt` file. Console output still present. |
| 22.4 | Bootstrap logger | Introduce startup error before host builds | Error logged to console + file (bootstrap logger), not to MongoDB (not yet configured). |
| 22.5 | Silent catch — AuthService | Submit invalid password-reset token | `LogWarning` entry: "Password-reset token validation failed" with exception stack trace in MongoDB `logs`. |
| 22.6 | Silent catch — ChatSessionRepository | Insert malformed BSON chat session doc, then fetch | `LogWarning` entry: "Malformed chat-session document for UserId=…" with `BsonSerializationException` in MongoDB `logs`. |
